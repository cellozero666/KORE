# ADR-002 - Companion Architecture

## Status

Accepted

---

## Contexto

O K.O.R.E. Companion é uma aplicação multiplataforma que integra interfaces React com recursos nativos do sistema operacional e comunicação com o K.O.R.E. OS.

Para manter uma arquitetura modular, segura e reutilizável, é necessário separar claramente as responsabilidades entre frontend e backend.

---

## Decisão

O Companion será dividido em duas camadas.

```text
React
    ↓
State Global
    ↓
Tauri
    ↓
Rust
    ↓
Swift (quando necessário)
```

O Backend será a única camada autorizada a:

- Acessar Serial USB.
- Acessar Bluetooth Low Energy (BLE).
- Comunicar-se com o K.O.R.E. OS.
- Consumir APIs externas.
- Gerenciar autenticação OAuth.
- Gerenciar Refresh Tokens.
- Persistir configurações.
- Atualizar o State Global.

O React será responsável apenas por:

- Exibir informações.
- Receber interação do usuário.
- Enviar solicitações ao Backend.
- Observar o State Global.

O React nunca deverá acessar diretamente hardware, APIs externas ou armazenar credenciais.

---

## Consequências

- Existe uma única fonte de verdade para os dados da aplicação.
- O frontend permanece desacoplado das integrações nativas.
- A lógica de negócio permanece centralizada no Backend.
- A arquitetura pode ser reutilizada pelas versões Windows, Linux, iOS e Android.
