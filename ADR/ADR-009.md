# ADR-009

## Título

Communication Architecture

---

## Status

Aprovado

---

## Contexto

O Companion é responsável por estabelecer a comunicação entre o computador e o firmware K.O.R.E. OS.

Essa comunicação deverá ser transparente para o restante da aplicação, independentemente do meio físico utilizado.

O React nunca deverá conhecer BLE, Serial ou qualquer protocolo de transporte.

Toda comunicação deverá ocorrer exclusivamente através da camada Backend (Rust + Tauri).

---

## Decisão

A arquitetura de comunicação será composta pelas seguintes camadas:

React

↓

Hooks

↓

Tauri Commands

↓

Communication Manager

↓

BLE ou Serial

↓

K.O.R.E. OS

---

## Backend

Toda a implementação deverá permanecer em:

src-tauri/src/communication/

Estrutura:

communication/

    mod.rs

    manager.rs

    ble.rs

    serial.rs

    handshake.rs

    protocol.rs

    types.rs

---

## Commands

Toda comunicação com o Frontend deverá ocorrer através de:

src-tauri/src/commands/

Inicialmente:

commands/

    connection.rs

Novos comandos deverão ser adicionados neste diretório.

---

## Meio de Comunicação

Ordem obrigatória:

1. BLE
2. Serial USB (fallback)

Caso BLE não esteja disponível, o Communication Manager deverá tentar automaticamente Serial.

O Frontend nunca deverá conhecer qual meio está sendo utilizado.

---

## BLE

A implementação oficial utilizará:

BLE = CoreBluetooth (Swift) no macOS.

Motivos:

- multiplataforma
- compatível com macOS
- integração direta com Rust
- evita dependência de Swift neste momento
- facilita futura expansão para Windows e Linux

---

## Serial

A implementação utilizará:

serialport

Já aprovado nas dependências do projeto.

---

## Handshake

Toda conexão deverá obrigatoriamente executar:

Companion

↓

PING

↓

Firmware

↓

KORE_COMPANION

Somente após sucesso do Handshake o estado Connected poderá ser publicado ao Frontend.

---

## Eventos

O Backend deverá emitir eventos Tauri para atualização do Frontend.

O React nunca realizará polling.

---

## Estado

O Backend é responsável por detectar alterações de conexão.

O AppStore apenas observa os eventos emitidos pelo Backend.

---

## Responsabilidades

React

- Interface
- Consumo do AppStore

Hooks

- Invocar Commands
- Atualizar Stores

Commands

- Bridge React ↔ Backend

Communication Manager

- Escolher BLE ou Serial
- Controlar conexão
- Gerenciar Handshake
- Emitir eventos

BLE

- Implementação Bluetooth

Serial

- Implementação USB Serial

Firmware

- Responder protocolo K.O.R.E.

---

## Consequências

O restante da aplicação permanece completamente desacoplado do meio físico.

Mudanças futuras na camada de comunicação não exigirão alterações no React.

Novos transportes poderão ser adicionados sem modificar a interface pública do Companion.
