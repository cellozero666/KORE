# ADR-005 - State Management

## Status

Accepted

---

## Contexto

O K.O.R.E. Companion integra múltiplos módulos, serviços externos e comunicação com o K.O.R.E. OS.

É necessário manter um estado único e consistente para toda a aplicação, evitando duplicação de dados e estados conflitantes.

---

## Decisão

O Companion utilizará um único State Global como fonte oficial da verdade.

Todo módulo do Backend será responsável por atualizar o State Global.

O React nunca deverá consultar diretamente:

- APIs externas.
- Hardware.
- Bluetooth Low Energy (BLE).
- Serial USB.
- Banco de dados.
- Arquivos de configuração.

O React deverá apenas:

- Observar o State Global.
- Atualizar a interface.
- Enviar ações ao Backend.

Toda alteração de estado deverá ser iniciada pelo Backend e refletida automaticamente na interface.

---

## Consequências

- Existe uma única fonte de verdade para toda a aplicação.
- O frontend permanece desacoplado das integrações externas.
- Elimina estados duplicados.
- Facilita testes, manutenção e escalabilidade.
- Permite reutilização da arquitetura em todas as plataformas suportadas.
