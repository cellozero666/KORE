# ADR-001 - Specification Driven Development

## Status

Accepted

---

## Contexto

O K.O.R.E. é um projeto de longa duração desenvolvido com auxílio de IA. Para manter consistência, reduzir perda de contexto e facilitar manutenção, o projeto adota uma metodologia baseada em especificações.

---

## Decisão

Todo o desenvolvimento seguirá Specification Driven Development (SDD).

Regras obrigatórias:

- Nenhuma implementação sem SPEC.
- Toda SPEC representa um único módulo.
- Todo módulo possui um Harness.
- Os módulos são implementados em ordem.
- Ao concluir um módulo, a implementação deve parar para testes e validação.
- O próximo módulo somente poderá iniciar após aprovação do usuário.

Fluxo:

SPEC

↓

Harness

↓

Implementação

↓

Testes

↓

Validação

↓

Próximo módulo

---

## Consequências

- Desenvolvimento incremental.
- Menor degradação de contexto.
- Arquitetura documentada antes da implementação.
- Melhor colaboração entre agentes de IA.
